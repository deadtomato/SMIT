• Разметка

    • 1. Выполнена HTML-разметка всех страниц проекта и всех элементов на этих страницах.
    • 2. Грубые ошибки в разметке отсутствуют.
    • 3. Документ проходит проверку на валидность validator.w3.org.
    • 4. В разметке отсутствует дублирование кода для одного и того же элемента, с помощью которого элемент отображается в разных местах страницы на разных версиях: мобильной, десктопной, планшетной. Этот критерий не касается элементов, которые скрываются или показываются в разных версиях.
    • 5. Отсутствуют типовые ошибки в разметке по методологии.
    • 6. Названия полей форм привязаны к своим полям с помощью <label>.
    • 7. У всех изображений размер прописан в теге <img>, у встроенных SVG-изображений размер прописан в теге <svg>
    • 8. Использовано минимально возможное количество HTML-элементов (нет лишних элементов).

  • Стилизация

    • 1. Раскладка блоков на странице сделана с помощью флексбоксов.
    • 2. В CSS отсутствует !important.
    • 3. Подключены правильные шрифты, их размеры, цвет и толщина равны соответствующим параметрам в макетах и техническом задании.
    • 4. Нестандартные шрифты подключены локально. Формат шрифтов должен быть woff и woff2.
    • 5. Указаны альтернативные варианты шрифта и тип семейства в конце перечисления font-family.
    • 6. При наполнении контентом (как в макете) элементы каждой версии страницы (мобильной, планшетной и десктопной) соответствуют макету.
    • 7. Для стилизации не использованы #id
    • 8. Для блока, у которого есть фоновое изображение, прописан фоновый цвет, который соответствует преобладающему цвету изображения (пока изображение не загружено, страница выглядит похоже на макет).
    • 9. Нет глобальных стилей тегов.

  • CSS-препроцессор

    • 1. Использован CSS-препроцессор.
    • 2. Код стилей должен быть разбит на несколько частей.

  • Адаптивность

    • 1. Выполнена вёрстка трёх состояний каждой страницы: мобильной, планшетной и десктопной.
    • 2. В разметке есть правильный вьюпорт тег.
    • 3. Для микросеток использованы флексбоксы.
    • 4. Логотип должен адаптироваться.
    • 5. Выполнено кадрирование контентных изображений.

  • Графика

    • 1. Использована векторная графика.

  • Сборка проекта

    • 1. Процесс сборки проекта настроен с помощью Gulp
    • 2. Все зависимости проекта должны быть указаны в файле package.json. Команда npm i должна установить всё необходимое для того, чтобы сборка проекта работала.
    • 3. Сборка проекта запускается командами npm run build и npm start
    • 4. В результате сборки должна получаться папка build со всеми необходимыми файлами.
    • 5. Папка build со всем её содержимым не должна попадать в Гитхаб.

  • Тестирование

    • 1. Вёрстка проходит тест на переполнение контентом.

  • Доступность

    • 1. У интерактивных элементов при нажатии или фокусе с клавиатуры есть активное состояние.
    • 2. Все интерактивные элементы имеют текстовое описание.

  • Оптимизация

    • 1. Использованы изображения в формате WebP.
    • 2. Использован векторный спрайт.
    • 3. Произведена оптимизация загрузки шрифтов.

  • Разное

    • 1. Вёрстка идентично отображается в последних версиях браузеров Chrome, Firefox, Safari, Edge, Internet Explorer.
    • 2. Единообразное написание и форматирование кода в HTML, файлах CSS-препроцессора и JavaScript (включая файлы автоматизации).
    • 3. Отсутствует транслит в названиях классов, атрибутах, переменных CSS-препроцессора, названиях примесей и так далее.
    • 4. Мобильное меню должно быть работоспособным при отключённом JavaScript.
    • 5. Проект соответствует техническому заданию.
    • 6. При взаимодействии с элементами (наведение, нажатие) ни сам элемент, ни окружающие его блоки не меняют своего положения.
