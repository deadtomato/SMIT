• 1. На сайте кликаем по ссылек “Котейка”

    Тут же можно нажать на ссылке “Макет”, чтобы посмотреть на макет сайта, который необходимо сверстать

    Обратите внимание, все версии проекта представлены на панели слева в разделе Pages

  • 2. На странице проекта можно ознакомиться с Техническим заданием и Критериями качества.

    Кроме того, на первой вкладке расположены Модули заданий, которые необходимо будет выполнять в той последовательности, в которой они представлены здесь. К заданиям мы еще вернемся, а пока нам надо сгенерировать мастер-репозиторий.

  • 3. Нажимаем на кнопку Создать проект, открывается модальное окно. Здесь надо ввести имя и фамилию

    Имя и фамилия должны быть реальные, прочее будет удалено!

    Жмем на кнопку Создать репозиторий. Появляется ссылка, кликаем!

  • 4. Попадаем на страницу с мастер репозиторием. Нам надо сделать форк! Нажимаем кнопку Fork, создается форк репозитория, и автоматически перенаправляет в наш аккаунт.

    Клонируем репозиторий на компьютер, нажимаем кнопке Clone or download, копируем URL и в консоли пишем:

    git clone <ссылка на форк>

  • 5. Теперь надо добавить ссылку на мастер-репозиторий:

    git remote add main <ссылка на мастер-репозиторий>

    Проверяем:

    git remote -v

  • 6. Можно выполнять задания. Создаем ветку первого задания:

    git checkout -b task-1

    После того как задание выполнено, надо закоммитеть изменения и запушить в форк.

    Можно добавлять и коммитить каждый файл, а можно все сразу, делайте как вам удобно.

    git add .

    git commit -m “Описание коммита”

    git push origin task-1

  • 7. Теперь в интерфейсе Github видим коммит, тут же кнопка создания пулреквеста, нажимаем, создаем и ждем проверки!

  • 8. Если есть замечания, необходимо их исправить, для этого возвращаемся в редактор.

    Надо убедиться, что мы на нужной ветке!

    Исправляем! И снова делаем коммит и пушим в форк.

    git add .

    git commit -m “Описание коммита”

    git push origin task-1

    При этом не надо создавать новый пулреквест, если запушили в ту же ветку из которой создавали пулреквест, изменения автоматически попадут в пулреквест.

  • 9. Когда все хорошо и пулреквест принят, а изменения смержены в мастер-репозиторий, надо забрать их и отправить в ветку master нашего форка!

    Идем в консоль. Переключаемся на ветку master:

    git checkout master

    Забираем изменения из мастер-репозитория:

    git pull main master

    И отправляем в форк:

    git push origin master

  • 10. Можно приступить к выполнению второго задания.

    Создаем ветку task-2:

    git checkout -b task-2

    Выполняем, коммит, пуш в форк в ветку task-2, делаем ПР и так по кругу!
