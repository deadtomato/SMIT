• 1. Общие технические требования

    • 1.1. Сетка: определена в макете.
    • 1.2. Адаптивность сетки: мобильная, планшетная и десктопная версии («фикс» или «резина»).
    • 1.3. Адаптивность графики: ретинизация, векторные изображения.
    • 1.4. Используемая методология: БЭМ.
    • 1.5. Используемый препроцессор: Sass.
    • 1.6. Используемый инструмент автоматизации: Gulp.
    • 1.7. Кроссбраузерность: Chrome, Firefox, Safari, Edge
    • 1.8. Обязательными к вёрстке являются все 3 страницы (главная страница, страница каталога, карточка номера), а так же 2 попапа (форма брони номера и сообщение об успешной отправки формы).

  • 2. Пожелания к поведению блоков

    • 2.1. Между версиями (мобильная, планшетная, десктопная) сетка может быть как резиновой, так и фиксированной.
    • 2.2. При фиксированной сетке контентная область центруется и не может быть уже макетной ширины. Фоны, которые упираются в края макета должны тянуться на всю страницу.
    • 2.3. Логотип на внутренних страницах — это ссылка на главную страницу.
    • 2.4. Мобильное меню может быть реализовано двумя способами: как без JS, так и с использованием JS.
    • 2.5. При реализации без использования JS главное меню в мобильной версии должно быть всегда открыто, а иконка с крестиком — скрыта.
    • 2.6. При реализации с использованием JS блок с главным меню в мобильной версии должен открываться при нажатии на иконку «гамбургера». Когда меню открыто, иконка «гамбургера» заменяется на крестик. При нажатии на иконку с крестиком меню закрывается.

  • 3. Форма

    • 3.1. Должны быть реализованы кастомные элементы форм.
    • 3.2. У полей ввода телефона и почты должны быть указаны соответствующие типы для удобного заполнения с телефона.
    • 3.3. При попытке отправить форму, в которой поля для ввода телефона или почты заполнены данными в неподходящем формате, появляется встроенное в браузер сообщение об ошибке.
    • 3.4. Кнопка «Забронировать» вызывает появление модального окна для оформления брони. При отключенном JS должен осуществляться переход на страницу оформления заказа (страницу оформления заказа реализовывать не нужно).
