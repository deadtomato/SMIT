• 1. Подготовка к работе

    • 1. Сделайте форк проекта
    • 2. Затем, клонируйте свой репозиторий (форк) проекта к себе на компьютер.
    • 3. Создайте ветку task-1 в склонированном проекте.
    • 4. В этой ветке инициализируйте npm-пакет.
    • 5. Установите gulp и плагины (gulp-sass, gulp-plumber, gulp-postcss, gulp-sourcemaps, autoprefixer,browser-sync)
    • 6. Создайте и сконфигурируйти gulpfile.js
    • 7. Напишите скрипты в packages.json
    • 8. Создайте структуру проекта в папке source. В пустых папках img, fonts и т.д. создайте файл README (БЕЗ РАСШИРЕНИЯ!), в котором укажите, что должно быть в этой папке. К примеру, в папке img должен лежать файл README с текстом: "Папка для изображений"
    • 9. Создайте в source файл index.html. Создайте базовую разметку. В body создайте заголовок первого уровня с текстом Hello world!
    • 10. Выполните тестовый запуск локального сервера, для проверки результата.
    • 11. Создайте пулреквест из ветки task-1 вашего форка в мастер-репозиторий.
    • Если требуются дополнительные изменения, то их нужно вносить в ветке из которой создан пулреквест. Новый пулреквест создавать не нужно. Все изменения автоматически появятся в уже созданном пулреквесте.

  • 2. Разметка

    Прежде чем приступить к заданию необходимо обновить ветку master

    • В вашей локальной копии переключитесь в ветку master

      git checkout master

    • Заберите изменения из мастер-репозитория

      git pull main master

      В main должна быть ссылка на мастер-репозиторий, Если ее там нет, добавьте:

      git remote add main <ссылка на мастер-репозиторий>

    • Отправьте изменения ветки master в ваш форк на Github

      git push origin master

    • Когда обновите master, создайте ветку для выполнения задания

      git checkout -b task-2
    • 1. Выполните разметку трёх страниц мобильной версии проекта. Разметка должна быть выполнена в соответствии с методологией БЭМ. (Не используйте обозначение мобильной версии -mobile в именах файлов или классов)

      Пример хороших названий страниц в проекте:

        cathotel
  |-source
  |--index.html
  |--catalog.html
  |--room.html
      На этом этапе нужна только HTML-разметка, без CSS. Изображения пока вырезать не нужно — используйте вместо них заглушки, например: <img src="https://via.placeholder.com/150x150">
    • 2. Создайте пулреквест из ветки task-2 вашего форка в мастер-репозиторий, чтобы отправить задание на проверку.

  • 3. Начало стилизации с препроцессором.

    Прежде чем приступить к заданию необходимо обновить ветку master

    • В вашей локальной копии переключитесь в ветку master

      git checkout master

    • Заберите изменения из мастер-репозитория

      git pull main master

      В main должна быть ссылка на мастер-репозиторий, Если ее там нет, добавьте:

      git remote add main <ссылка на мастер-репозиторий>

    • Отправьте изменения ветки master в ваш форк на Github

      git push origin master

    • Когда обновите master, создайте ветку для выполнения задания

      git checkout -b task-3
    • 1. Определите Шрифты в макете, скачайте и добавьте в папку для шрифтов в проекте.
    • 2. В папке sass создайте файлы: style.scss, fonts.scss, globals.scss.
    • 3. В fonts.scss подключите шрифтовые гарнитуры.
    • 4. В globals.scss напишите глобальные свойства.
    • 5. Для проверки в файле source/sass/style.scss добавьте CSS-код: 
          body { 
      background: red; 
    }
      Запустите сервер.

      В папке css должны появиться 2 файла style.css и файл style.css.map

    • 6. Далее необходимо подключить style.css на свои страницы (не забудьте сохранить изменения).

      Фон страницы в браузере должен стать красным.

    • 7. Создайте структуру препроцессорных файлов:
      • каждому БЭМ-блоку должен соответствовать отдельный препроцессорный файл
      • файлы БЭМ-блоков импортируются в файле source/sass/style.scss
      • также в файле source/sass/style.scss импортируются файлы шрифтов и глобальных стилей
      • вспомогательные файлы с переменными, примесями и так далее создавайте на своё усмотрение
    • 8. Начните вёрстку вашего проекта с помощью препроцессора:
      • задайте фоновые цвета и фоновые изображения для блоков
      • параметры текста: шрифты, размеры шрифта, межстрочные расстояния, цвет текста
      • добавьте внутренние и внешние отступы и обводки для крупных сеточных блоков
      Мелкие декоративные элементы можно не верстать.
    • 9. Создайте пулреквест из ветки task-3 вашего форка в мастер-репозиторий, чтобы отправить задание на проверку.
